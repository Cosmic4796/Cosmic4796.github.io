3. Data Handling

What Data We Store

Fun.Bot is designed to store only the absolute minimum non-personally identifiable information essential for its core functionality and the delivery of its intended features:

  • User IDs: Your unique Discord User ID is stored to facilitate the management of in-bot economy balances, progression levels, game status, and user-specific configurations. This ID is a fundamental identifier provided by the Discord platform.
  • Server (Guild) IDs: Server identifiers are stored to manage configurations specific to each Discord server (e.g., custom command prefixes, enabled/disabled features, welcome message settings).
  • Channel IDs & Role IDs: These identifiers are only stored when a server administrator designates and configures them explicitly for specific bot functionalities, such as automated logging channels or role assignment mechanisms.
  • Command Usage Data: We collect aggregated and anonymized statistical data concerning command invocation frequency, feature access patterns, and interaction timestamps. This data is exclusively used for internal analytics, performance optimization, troubleshooting, and future feature planning. This data is not traceable to individual users and cannot be used for personal identification.
  • Game Data: For interactive features such as the economy system, mini-games, or leveling systems, we store relevant gameplay data, including virtual currency balances, virtual inventory items, scores, and progression metrics. This data is directly linked to your Discord User ID to ensure a seamless and persistent in-bot experience.

It is critical to emphasize that Fun.Bot does NOT collect or retain any sensitive personally identifiable information (PII), including but not limited to, your real name, email address, IP address, physical location, or private direct messages. All data acquired is either directly provided by the Discord API or generated through your voluntary interactions with the bot's publicly available commands.

Data Retention Policies

  • Active User Data: Information associated with users who have actively engaged with the bot within the preceding 12-month period will be retained until such a time as a deletion request is formally submitted by the user, or the user transitions to an inactive status.
  • Inactive User Data: User-specific data (primarily Discord User IDs and their linked in-bot economy/game data) belonging to users who have exhibited no interaction with Fun.Bot for a continuous duration of 1 year will be automatically purged from our databases.
  • Server Data: Server configuration data will be maintained for the entire duration that Fun.Bot remains an active member of the respective Discord server. Should the bot be removed from a server, all associated server data will be irrevocably deleted.

1. Information We Collect

Fun.Bot is committed to user privacy and collects only the absolutely essential information required for its seamless operation and to deliver its designed features:

  • Discord User Data: We collect your unique Discord User ID. For certain visualizations within the bot (e.g., leaderboards, user profiles), we may access your public username, discriminator, and avatar URL via the Discord API. This information NOT persistently stored on our servers unless explicitly required for a specific bot feature (e.g., a custom profile background feature).
  • Discord Server Data: We collect Server (Guild) IDs to manage server-specific configurations (such as custom command prefixes, enabled/disabled commands or automated welcome messages) and to associate data with the correct server environment. Channel IDs and Role IDs are only collected when a server administrator configures them explicitly for specific functionalities (e.g., logging channels, automatic role assignments).
  • Usage Data: We gather anonymized and aggregated statistical data concerning command executions, feature access frequency, and interaction timestamps. These data are exclusively utilized for internal analytics, performance enhancement, debugging and strategic planning for future bot improvements. These data are not linked nor can they be utilized to identify personal users.
  • Game Data: For interactive features such as the in-bot economy, mini-games or leveling systems, we store relevant gameplay data, including virtual currency balances, virtual inventory items, game scores and progression milestones. These data are directly linked to your Discord User ID to ensure the continuity of your in-bot experience.

Fun.Bot maintains a strict policy against collecting any sensitive personal information, including, among others, your IP address, real name, email address, physical address, or private direct messages. All data obtained is either directly furnished by the Discord API or is generated through your voluntary interactions with the bot's publicly accessible commands.

The information meticulously collected by Fun.Bot is exclusively employed for the following defined purposes:
